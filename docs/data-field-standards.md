# 匠人程数据字段规范

更新时间：2026-06-21

## 目标

这份规范用于三件事：

- 老师继续用 Excel 时，字段名称尽量统一。
- 系统导入 Excel 时，减少人工解释和二次清洗。
- 后面接阿里云数据库时，字段能直接落表，不反复返工。

## 通用命名规则

### 人名

- 老师姓名、学生姓名、家长姓名都用中文真实姓名。
- 同一个人不要混用简称、昵称、微信名。
- 老师姓名以人事系统全员名单为准。

### 日期

- 日期统一写成 `2026-06-21`。
- 月份统一写成 `2026-06`。
- 不写 `6.21`、`6月21号`、`明天`、`周六` 这类相对日期。

### 时间

- 时间统一写成 `18:30`。
- 时间段拆成两个字段：`start_time`、`end_time`。
- 不写 `晚上6点半`、`六点半到八点`。

### 手机号

- 家长手机号、员工手机号只保留数字。
- 同一个手机号视为同一个客户/家长，导入时会用于去重。

### 金额

- 金额只写数字，单位默认为元。
- 不写 `￥`、`元`、`,`。

### 状态

状态字段尽量使用系统给出的固定选项。暂时不确定时写 `待确认`，不要自己发明新状态。

## 系统主键口径

### 员工

- 主识别字段：`teacher_name` / `employee_name`
- 登录字段：`username`
- 手机号：`phone`

### 学生/家长

- 学生主识别：`student_name + parent_phone`
- 家长主识别：`parent_phone`
- 同一个手机号导入多次时，系统应合并历史跟进记录。

### 班级

- 班级主识别：`class_name + subject + grade + term`
- 暑假班建议 term 写 `2026-summer`。
- 平时班建议 term 写 `2026-06` 或 `2026-fall`。

### 课程/课次

- 课次主识别：`course_date + start_time + end_time + teacher_name + class_name`
- 教室允许后补，但日期、时间、老师、班名必须尽量准确。

## 导入优先级

1. 先保证姓名、手机号、日期、金额这些基础字段准确。
2. 班级名、教室、备注可以后补，但要尽量保留原始描述。
3. 系统识别不出的字段进入待确认，不直接丢弃。
4. 后面每次老师发新表，都按这份规范逐步靠拢。

## 当前最需要老师配合规范的地方

- 老师姓名不要混用简称。
- 班级名称尽量固定，例如 `初一数学暑假A班` 不要一会儿写 `初一A`、一会儿写 `七数A`。
- 课程日期必须写完整日期。
- 教室可以空，但不要用无法判断的符号代替。
- 试听、补课、一对一、答疑、免费课要写在 `course_type` 或 `notes` 里。
